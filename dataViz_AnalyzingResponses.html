Analyzing responses to rate the usefulness of a feature of a software tool¶

Objective of the survey:¶

The objective of the survey is to solicit feedback from subjects in order to evaluate the responses to rate the usefulness of a software tool. This helps us understand usefulness of our software tool to users and also the shortcoming of the software tool inturn helping us understand the areas of improvement to better the user experience using the software.

  • Understand the effectivness and expressiveness of the Data/Task Abstraction component.
  • Understand the effectiveness and epressiveness of the Visual Encoding/Interaction Idiom component.
  • Process collected responses.

Questionaire Response collection¶

As we can see from the dataset the entire survey has been conducted using a 5 point scale rating system. This can be considered as a likert scale rating system.

25 subjects were presented with 11 questions to understand different areas of the software usage and respective responses were recorded into a matrix of 25 X 11 where each row represented different user and each column corresponds to each question that was presented to the user.

I have renamed the columns as Q1, Q2, Q3......Q11 and also have renamed the first column as "Respondants". The dataset before and after the renaming of columns are shown below:

Data Pre-Processing¶

The current format of dataset(one-row -per-respondent format) does not allow to derive very interesting patterns and also to create good visualization. So in order to improve the dataset format we have to understand the context of the questions so that we can create groupings, labels etc.

It is known that:

  • Questions 1, 2 and 4 are in the same group. Each of these three questions asks the subjects to evaluate whether one (of the three) aspects of a visualization system was useful or not.
  • Questions 5, 6 7 and 8 ask the subjects to provide a rating of one of the four important visualization components to ensure each component did help him/her understand a particular feature.
  • Questions 10 and 11 ask the subjects for their experience in using the system to learn and to improve their previous study.
  • Question 12 asks the subjects to report how long he/she took to understand the concept by using the visualization tool. The answers are range values as people may not be able to point out exactly how many minutes they have spent on the visualization system. Higher values mean longer time spent on the system.
  • Question 13 asks how many times a subject used the visualization system in his/her study. The answers indicate ranges. Again, higher values means more times.

From the above known facts we can see that there are 5 high level groupings of the questions and we can name each of these groupings.

  • Group1 (Q1, Q2, Q4) - Aspect of Visualization [here the rating imply how useful an aspect of Viz was]
  • Group2 (Q5, Q6, Q7, Q8) - Visualization Components [here the rating imply how each component helped user understand a feature]
  • Group3 (Q10, Q11) - Experience in using the system. [here the ratings imply how much experience user has using the system]
  • Group4 (Q12) - Time to understand the concept [here the ratings imply time taken by user to understand a concept]
  • Group5 (Q13) - Times visualization system was used by subject

We move ahead to labelling the ratings based on different groups. Most of the labels below are selected from recommended likert scale labelling obtained from a study by Iowa State University.

  • For Group1 and Group2 the suitable labels for the ratings chosen are:

      1) Strongly Disagree
  2) Disagree
  3) Neutral
  4) Agree
  5) Strongly Agree

  • For Group3:

      1) Novice
  2) Advanced beginner
  3) Competent
  4) Proficient
  5) Expert

  • For Group4:

      1) Very Short
  2) Short
  3) Moderate
  4) Long
  5) Very Long

  • For Group5:

      1) Never
  2) Rarely
  3) Normal
  4) Sometimes  
  5) Most of the time

Analysis:¶

In order to clearly visualize how software tool was useful for users and also to see a general user sentiment towards different areas of software tool I chose to use a Divergent Stacked Bar (Similar to a modified Gantt chart).

A divergent stacked bar chart is a juxtapose of a dual axis stacked bar chart and a bubble plot (showing the average rating for a given question and aspect of visualizations). The plot is as shown below:



To obtain the plot we had first derive some calculated Fields as shown below:

  • First we divide the responses into "Positive Responses" and "Negative Rsponses".
  • To calculate the positive responses, we have assumed any rating above "3" is considered positive and the rest as negative resposnses.
  • We then calculate the average ratings for a given question show the value as circle plot(in few tools it is used as bubble plot).

The idea is based on sentiment analysis, where we divide the users into "Promoters"(users who rate high) and "Detractors"(users who rate low to neutral) and calculate "Percentage of positive responses" and "Percentage of negative responses".

In our case, Groups 1 and 2 indicate sentiment of the user towards different aspects of the visualization and Visualization components used.

1) It is clear from the above plot that users generally showed a positive response towards different aspects of the visualization system. However Q2 shows that users had a mixed feeling towards the second aspect of the visualization system used and hence needs some improvements to be made to better the user experience.

2) Again for Group2 Questions, we see a generally positive response. But we see that Q6 (or Second Visualization component) showed poor response with an Average rating of 0.7. Hence we need to massively improve the second visualization component.

3) We see from Group3 Questions that most of the users who were part of the survey had very good experience using the system.

4) The most interesting insight was that even though most users had lot of experience using the system, the "Number of times the visualization system was used" was very low inturn a negative response to the system usage. This seem to be contradictory given that the "Time taken to understand a concept also seems to show that most users understand the concept in short time. Based on the given data it is difficult to show why we are seeing this reduction in users adapting the system as most features in our dataset show positive response.

How different are the user responses given the questions posed to the users?¶
MANOVA - Multivariate analysis of variance¶

In order for us to use MANOVA we need to check if the data features have a good correlation as a MANOVA model on uncorrelated dimensions to not yield accurate results. To do that, I have plotted a correlation plot(using pearson method) and we can see that most features of the dataset show independence. Only Q8 and Q7 act as dependent variable given the independent variable "Responsents".

Null Hypothesis: The responses for Q7 and Q8 (third and fourth Visualization component) are same across users(independent variable)¶
Alternative hypothesis: Q7 and Q8 (third and fourth Visualization component) are significantly different across Respondents.¶

Are there any clusters or groupings among the responses from subjects?¶

This can be answered using a Cluster map which is a juxtaposed version of a dendogram and heatmap plots.

  • A dendrogram is a tree diagram frequently used to illustrate the arrangement of the clusters produced by aglomerative hierarchical clustering.
  • A heat map is a graphical representation of data where the individual values contained in a matrix are represented as colors.

Using a cluster map we can clearly see the ratings of users across the different areas of study of the software tools.

  • From the plot we can see that first,second aspect and second visulization component was the most useful to the users.
  • We can also see that most of the respondents have high experience in using the system. But still resulting in low adaptation of the system.
  • We can again see the contradiction indicated in previous sections that even though the learning time to understand a concept is low, the system adaption into users study is very low.
